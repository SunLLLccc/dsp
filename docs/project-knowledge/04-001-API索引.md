## 全局约定

- **统一请求报文**：`ApiRequest<T>`（dsp-common/model/ApiRequest.java），含 `head: RequestHead` + `requestData: T`。`RequestHead` 字段：`token`/`appId`/`timestamp`（均 `@NotBlank`）、`traceId`、`transno`。**仅 dsp-data-service 的 `/dsp/api/**` 强制使用此报文**。
- **统一响应报文**：`ApiResponse<T>`（dsp-common/model/ApiResponse.java:6），结构 `{ head: ResponseHead, code, message, data }`，由 `ApiResponse.success(transno, traceId, data)` / `ApiResponse.error(...)` 工厂构造。**dsp-admin-service 与 dsp-offline-service 复用此结构**（admin 多数方法 transno 传业务语义常量如 `"LOGIN"`/`"EXPORT_LIST"`）。
- **错误码**：`ErrorCode` 枚举（dsp-common/enums/ErrorCode.java），如 `0000 成功`、`4001 TOKEN_MISSING`、`4003 ACCESS_DENIED`、`4004 INTERFACE_NOT_FOUND`、`4100 BAD_REQUEST`。
- **鉴权**：三服务各异——dsp-data-service 用 `JwtAuthAspect`（`/dsp/api/**`）；dsp-offline-service 用 `JwtAuthInterceptor`（`/dsp/offline/**`）；dsp-admin-service 用 `AdminAuthInterceptor` + `@RequireRole` 注解 + `RoleCheckAspect`（`/dsp/admin/**` 除 `/login`）。
- **全局异常处理**：各服务有 `GlobalExceptionHandler`（`@RestControllerAdvice`），统一转 `ApiResponse.error`。
- **无 API 版本前缀**：路径以 `/dsp/{api|offline|admin}/` 三段式区分服务与领域，无 `/v1` 等版本号。

## 调用关系总览

### 主干调用链

1. **外部数据查询链**：
   `外部应用 → [POST /dsp/api/{transno}] DataApiController.query → XmlConfigCacheManager.get(SPI 同模块) → XmlEngine.executeWithConfig → 多类型数据源（MySQL/MongoDB/Dubbo/HTTP）`

2. **在线导出链**：
   `外部应用 → [POST /dsp/api/{transno}/export] DataApiController → ExportService.onlineExport → DataQueryService.execute(SPI) → XmlEngine.execute`

3. **离线导出链**：
   `外部应用 → [POST /dsp/offline/export] OfflineExportController → ExportService.submitOfflineExport → (异步) executeOfflineExport → DataQueryService.execute(SPI)`

4. **管理端典型链**（前端 → admin-service）：
   `dsp-admin-web (axios, api/index.js) → Vite 代理 /dsp/admin → :8082 → AdminAuthInterceptor(鉴权) → XxxController → XxxService(dsp-core) → MyBatis-Plus Mapper → DB`

5. **配置变更失效缓存链**（SPI 解耦）：
   `前端 → InterfaceAdminController/TemplateAdminController → InterfaceVersionServiceImpl/TemplateServiceImpl(dsp-core) → XmlConfigCacheInvalidator.invalidate(SPI) → XmlConfigCacheManager(dsp-engine) 失效缓存`
   `前端 → ConfigImportExportController(dsp-admin-service) → XmlConfigCacheInvalidator.invalidate(SPI) → XmlConfigCacheManager(dsp-engine)`

6. **缓存定时刷新链**：
   `Spring @Scheduled → CacheRefreshScheduler.refreshAll → CacheLoadStrategy.loadActiveTransnos → XmlConfigCacheManager.refreshAll`

### 跨服务调用关系

- **3 个服务间无 RPC 调用**（已核实无 `@DubboService`/`@DubboReference`）。
- 服务间解耦方式：dsp-core / dsp-engine 作为 jar 被 3 个服务依赖（共享 Service/Engine 实现）；SPI 接口（dsp-common 定义）在编译期解耦 dsp-core 对 dsp-engine 的依赖。
- 唯一的跨服务运行期交互：外部应用分别调用 data-service（查询）与 offline-service（导出），offline-service 的导出逻辑复用 dsp-core 的 `ExportServiceImpl`（同 jar，非跨进程）。
