# DSP Domain Context

DSP 是一个面向零代码接口发布的数据服务平台。本文件只记录领域词汇，不记录实现方案或待办事项。

## Language

**联机接口**:
DSP 对外提供的在线数据查询接口，由接口元信息、输入输出 Schema 和 XML DSL 模板共同定义。
_Avoid_: API、接口、服务接口

**接口配置**:
描述一个联机接口可发布内容的配置集合，包含接口元信息、Schema 和 XML DSL 模板。
_Avoid_: 配置文件、接口 JSON

**Text2API**:
从自然语言或需求文档生成 DSP 联机接口配置的能力，目标产物是可被接口管理导入或发布的接口配置。
_Avoid_: 代码生成、接口生成器

**Text2SQL**:
Text2API 流程中把已确认的接口意图和输入输出转换为查询 SQL 的步骤。
_Avoid_: SQL 生成、自然语言转 SQL

**表结构依据**:
Text2SQL 生成 SQL 前必须具备的库表、字段、关联关系或数据源元数据信息。
_Avoid_: 猜测字段、模型常识

**数据源元数据**:
从已配置数据源读取的结构信息，包括库表、字段、类型、主键/索引和注释，不包含业务数据行。
_Avoid_: 数据抽样、查询数据

**模板驱动 XML 生成**:
Text2API 中先选择既有 XML DSL 模板再填充生成接口 XML 的方式，禁止 AI 脱离模板自由编写 XML。
_Avoid_: 自由 XML 生成、随写 XML

**修正记录**:
Text2API 中用户指出错误后沉淀的纠错信息，包含错误点、用户纠正、AI 修改结果和可复用经验。
_Avoid_: 聊天记录、反馈

**项目问答**:
智能助手面向 DSP 项目知识的问答能力，优先基于本地文档或代码检索结果回答。
_Avoid_: 普通聊天、客服问答

**源码兜底检索**:
项目问答中当本地文档不足以回答具体实现问题时，对项目源码进行受限检索来补足依据。
_Avoid_: 全量代码索引、源码 RAG

**Text2API 草稿**:
Text2API 流程中尚未最终导入或发布的中间产物集合，包括需求原文、接口输入输出、SQL、XML 和导入 JSON。
_Avoid_: 临时数据、会话缓存

**需求文档**:
用户提供给 Text2API 的接口需求说明，一期限定为 Markdown 或 HTML 格式且不超过 30KB。
_Avoid_: 附件、说明文档、PRD
